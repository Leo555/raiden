# Raiden Storm 游戏完善方案 - 产品需求文档 (PRD)

## 1. 项目信息

**语言**: 中文  
**技术栈**: HTML5 Canvas + ES Modules + Web Audio API（保持现有架构）  
**项目名称**: raiden_storm_enhancement  
**原始需求**: 为已完成的 Raiden Storm 竖版射击游戏制定完善方案，增加新内容、优化体验、提升可玩性。

---

## 2. 产品定义

### Product Goals（产品目标）

1. **提升游戏深度与重玩价值**：通过新增敌人类型、Boss 攻击模式、挑战模式等，让玩家有持续挑战和探索的动力
2. **增强视听沉浸感**：优化视觉效果和音频体验，让玩家获得更华丽的弹幕射击体验
3. **建立长期留存机制**：通过成就系统、皮肤解锁、排行榜等社交/成长系统，提升玩家粘性

### User Stories（用户故事）

1. **作为玩家**，我希望遇到更多样的敌人类型，这样游戏不会过于单调，每次游玩都有新鲜感
2. **作为核心玩家**，我希望 Boss 有更多攻击模式，这样挑战性更强，需要我不断调整策略
3. **作为休闲玩家**，我希望有成就系统和皮肤解锁，这样我能感受到成长和收集的乐趣
4. **作为竞技玩家**，我希望有在线排行榜和挑战模式，这样我能和他人比拼成绩
5. **作为视觉党**，我希望看到更华丽的爆炸效果和动态背景，这样游戏体验更爽快

## 3. 技术规范

### Requirements Pool（需求池）

#### **P0 - Must Have（核心玩法扩展）**

| 编号 | 需求 | 说明 | 优先级 |
|------|------|------|--------|
| P0-1 | 新增 3 种敌人类型 | 高速敌机（直线快速突进）、旋转敌机（边移动边旋转射击）、自爆敌机（接近玩家后爆炸） | 高 |
| P0-2 | 新增 Boss 攻击模式 | 每个 Boss 新增 2-3 种攻击模式（弹幕模式、激光攻击、追踪导弹） | 高 |
| P0-3 | 新武器系统 | 激光武器（持续伤害）、追踪弹（自动追踪敌人）、爆炸弹（范围伤害） | 高 |
| P0-4 | 更多道具类型 | 时间减缓（减速敌人子弹）、全屏清敌（清屏炸弹）、无敌护盾（短时间无敌） | 高 |

#### **P1 - Should Have（体验优化）**

| 编号 | 需求 | 说明 | 优先级 |
|------|------|------|--------|
| P1-1 | 视觉效果升级 | 更华丽的爆炸粒子效果、背景动态元素（漂浮的云/陨石）、关卡过场动画 | 中 |
| P1-2 | 音频体验优化 | 增加更多音效（敌人爆炸、道具拾取、Boss 警告）、动态音乐系统（根据战斗强度切换） | 中 |
| P1-3 | 成就系统 | 达成特定条件解锁成就（连续擦弹 50 次、无伤通关、收集 1000 金币等），共 20+ 成就 | 中 |
| P1-4 | 挑战模式 | 限时挑战（60 秒内击破最多敌人）、生存模式（存活尽可能长时间） | 中 |

#### **P2 - Nice to Have（长期功能）**

| 编号 | 需求 | 说明 | 优先级 |
|------|------|------|--------|
| P2-1 | 飞机皮肤系统 | 解锁不同外观和属性（速度型、防御型、攻击型），通过金币或成就解锁 | 低 |
| P2-2 | 在线排行榜 | 使用 Firebase 或类似服务存储全球高分排行榜，显示排名和玩家名称 | 低 |
| P2-3 | 3D 音效定位 | 根据敌人位置调整音效左右声道，增强沉浸感 | 低 |

### UI Design Draft（UI 设计稿）

#### **新增 UI 元素**

1. **成就通知弹窗**
   - 位置：屏幕顶部中央，向下滑出
   - 内容：成就图标 + 名称 + 描述
   - 动画：0.5 秒滑入，停留 3 秒后淡出
   - 样式：半透明黑色背景 + 金色边框 + 图标发光效果

2. **武器切换指示器**
   - 位置：屏幕右侧，玩家飞机上方
   - 内容：当前武器图标 + 等级（1-5 级）
   - 样式：小图标 + 等级数字，武器切换时闪烁提示

3. **Boss 血条**
   - 位置：屏幕顶部，居中
   - 内容：Boss 名称 + 血条（彩色渐变）
   - 样式：红色到黄色的渐变条，受击时闪烁白色

4. **挑战模式选择界面**
   - 位置：主菜单新增"挑战模式"按钮
   - 内容：两个选项卡片（限时挑战、生存模式）
   - 样式：卡片式设计，显示最佳成绩

5. **成就系统界面**
   - 触发：主菜单新增"成就"按钮
   - 布局：网格布局，3 列 × 7 行（21 个成就）
   - 样式：未解锁为灰色图标，已解锁为彩色 + 描边发光

6. **时间减缓特效**
   - 触发：拾取时间减缓道具后
   - 效果：屏幕边缘泛蓝光 + 敌人子弹拖尾特效
   - 持续时间：5 秒

### Open Questions（待确认问题）

1. **技术可行性**
   - 现有 Canvas 架构是否能支持 3D 音效定位？是否需要引入 Web Audio API 的 PannerNode？
   - 在线排行榜是否需要用户登录系统？还是仅记录昵称 + 分数？

2. **游戏平衡性**
   - 新武器（激光、追踪弹、爆炸弹）如何平衡？是否需要进行伤害测试？
   - 自爆敌机的伤害值应该是多少？是否应该让玩家有反应时间？

3. **美术资源**
   - 新敌人、Boss 攻击模式、武器效果是否需要新的美术素材？还是用程序化生成？
   - 成就图标是否需要设计师设计？还是使用 emoji 或简单几何图形？

4. **开发优先级**
   - P0 需求的 4 个功能是否应该分批次开发？还是一次性完成？
   - 挑战模式和成就系统是否可以复用现有代码框架？

5. **性能优化**
   - 新增更多粒子效果后，是否需要进行性能优化（对象池、离屏渲染）？
   - 移动端是否能流畅运行新增的视觉效果？

## 4. 实施建议

### 开发阶段划分

**第一阶段（2 周）**：P0 核心玩法扩展
- 新增 3 种敌人类型
- 新增 Boss 攻击模式
- 新武器系统
- 更多道具类型

**第二阶段（1 周）**：P1 体验优化
- 视觉效果升级
- 音频体验优化
- 成就系统
- 挑战模式

**第三阶段（1 周）**：P2 长期功能 + 优化
- 飞机皮肤系统
- 在线排行榜（可选）
- 性能优化 + 测试

### 技术风险

1. **Canvas 性能**：新增大量粒子和敌人可能影响性能，需使用对象池和空间分割优化
2. **音频兼容性**：Web Audio API 在移动端可能有延迟，需测试不同设备
3. **在线功能**：如果需要在线排行榜，需考虑后端服务和 API 设计

---

## 5. 成功指标

- **玩家留存率**：次日留存 > 40%，7 日留存 > 20%
- **游戏时长**：平均单次游戏时长 > 10 分钟
- **成就解锁率**：至少 50% 的玩家解锁 5 个以上成就
- **挑战模式参与率**：至少 30% 的玩家尝试挑战模式
